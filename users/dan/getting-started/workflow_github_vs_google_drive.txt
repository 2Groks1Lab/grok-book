# Workflow: GitHub vs Google Drive

## Core Principle
- **GitHub** = Working drafts + revision history
- **Google Drive** = Finalized / archived work

## Detailed Workflow

### 1. Draft Phase (GitHub)
- All active work, drafts, and in-progress content lives on **GitHub**.
- GitHub is used specifically because it provides **revision control** and **reversion** capabilities.
- This is critical for X (Twitter) publication workflows where we may need to roll back changes.
- Both Dan and Cody can troubleshoot, make manual edits, or review history directly in GitHub.

### 2. Finalization Phase (Google Drive)
- Once content is **finalized**, polished, or no longer actively being edited, it is moved to **Google Drive**.
- Google Drive serves as the long-term organized archive and backup location.
- Local copies on devices are deleted after successful upload to Google Drive to keep storage clean.

## Summary Table

| Stage          | Location       | Purpose                          | Key Feature          |
|----------------|----------------|----------------------------------|----------------------|
| Draft / Active | GitHub         | Daily work, revisions, collaboration | Revision control & reversion |
| Final / Archive| Google Drive   | Long-term storage, organization     | Backup + accessibility |

## Notes
- This workflow prioritizes revision safety during the creative/drafting phase.
- It keeps long-term storage clean and well-organized.
- Local device storage is treated as temporary working space only.
